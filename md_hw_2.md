### Конфигурация HAProxy

Использовался следующий конфигурационный файл:
```
global
	log /dev/log	local0
	log /dev/log	local1 notice
	chroot /var/lib/haproxy
	stats socket /run/haproxy/admin.sock mode 660 level admin expose-fd listeners
	stats timeout 30s
	user haproxy
	group haproxy
	daemon

	ca-base /etc/ssl/certs
	crt-base /etc/ssl/private

        ssl-default-bind-ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES256-GCM-SHA384
        ssl-default-bind-ciphersuites TLS_AES_128_GCM_SHA256:TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256
        ssl-default-bind-options ssl-min-ver TLSv1.2 no-tls-tickets

defaults
	log	global
	mode	tcp
	option	tcplog
	timeout connect 5000
	timeout client  50000
	timeout server  50000

listen stats
        bind :888
        mode http
        stats enable
        stats uri /stats
        stats refresh 5s
        stats realm Haproxy\ Statistics

listen web_tcp
        bind :1325
        mode tcp
        balance roundrobin

        server s1 127.0.0.1:8888 check inter 3s
        server s2 127.0.0.1:9999 check inter 3s
```


### Пояснение конфигурации HAProxy
`Секция global`

Секция global задает общие параметры работы HAProxy:

- логирование
- запуск в режиме демона
- пользователя и группу, от которых работает служба
- путь к chroot-окружению
- параметры SSL
- сокет администрирования

Эта секция отвечает за глобальные настройки самого сервиса.

`Секция defaults`

В секции defaults задаются параметры по умолчанию для всех остальных секций.

Ключевой строкой здесь является:
```
mode tcp
```
Она означает, что HAProxy работает в режиме TCP, то есть на 4 уровне модели OSI.
В этом режиме балансировщик не анализирует HTTP-заголовки и содержимое веб-запросов, а просто распределяет TCP-соединения между backend-серверами.

Также используется:
```
option tcplog
```
Эта директива включает логирование TCP-соединений.

`Секция listen stats`

Секция listen stats используется для отображения веб-страницы статистики HAProxy.
```
listen stats
        bind :888
        mode http
        stats enable
        stats uri /stats
```
Она не участвует в самой балансировке, но позволяет просматривать статистику работы HAProxy через браузер.

`Секция listen web_tcp`

Это основная секция, которая реализует балансировку нагрузки для задания.
```
listen web_tcp
        bind :1325
        mode tcp
        balance roundrobin

        server s1 127.0.0.1:8888 check inter 3s
        server s2 127.0.0.1:9999 check inter 3s
```
Что здесь происходит:

- bind :1325 — HAProxy слушает входящие подключения на порту 1325\
- mode tcp — работа выполняется на 4 уровне
- balance roundrobin — новые подключения распределяются между серверами по очереди
- server s1 127.0.0.1:8888 — первый backend-сервер
- server s2 127.0.0.1:9999 — второй backend-сервер
- check inter 3s — HAProxy периодически проверяет доступность серверов
- Проверка работы балансировки

После настройки конфигурации выполнялась проверка корректности файла:
```

```
Затем служба HAProxy была перезапущена:

```
sudo systemctl status haproxy
```
После этого была выполнена проверка доступности порта, который слушает HAProxy:

### 4. Настройка HAProxy

Для выполнения задания был настроен HAProxy в HTTP-режиме, то есть на 7 уровне.
В конфигурации было реализовано:

- прослушивание входящих HTTP-запросов на порту 80;
- проверка заголовка Host;
- разрешение обработки только запросов к example.local;
- запрет всех остальных HTTP-запросов;
- балансировка между тремя backend-серверами по алгоритму Weighted Round Robin;
- назначение весов 2, 3 и 4.

Использовался следующий конфигурационный файл /etc/haproxy/haproxy.cfg:

```
global
    log /dev/log local0
    log /dev/log local1 notice
    daemon

defaults
    log     global
    mode    http
    option  httplog
    timeout connect 5s
    timeout client  30s
    timeout server  30s

frontend http_front
    bind *:80

    acl host_example hdr(host) -i example.local
    http-request deny deny_status 403 unless host_example

    default_backend python_back

backend python_back
    balance roundrobin
    option httpchk GET /

    server py1 127.0.0.1:8081 check weight 2
    server py2 127.0.0.1:8082 check weight 3
    server py3 127.0.0.1:8083 check weight 4
```

В данной конфигурации:

- mode http переводит HAProxy в режим работы на 7 уровне;
- acl host_example hdr(host) -i example.local проверяет, что запрос адресован нужному домену;
- http-request deny deny_status 403 unless host_example запрещает все запросы, которые пришли не на example.local;
- balance roundrobin включает алгоритм распределения запросов;
- параметр weight задаёт относительный приоритет каждого backend-сервера.

После настройки конфигурация была проверена и сервис был запущен.
