- На проверку направить скриншот с командой и результатом ее выполнения

### Выполнение задания 1:
   
Для создания зеркальной копии домашней директории пользователя в каталог /tmp/backup была использована команда:

```
rsync -avc --delete --exclude='.*/' "$HOME/" /tmp/backup/
```

Что делает команда:

- `-a` — включает архивный режим, то есть рекурсивное копирование с сохранением структуры и атрибутов файлов;
- `-v` — выводит подробную информацию о процессе копирования;
- `-c` — включает сравнение файлов по хэш-сумме, даже если размер и время изменения совпадают;
- `--delete` — удаляет из каталога /tmp/backup файлы и каталоги, которых больше нет в исходной директории, благодаря чему создаётся именно зеркальная копия;
- `--exclude='.*/'` — исключает из синхронизации все скрытые директории, начинающиеся с точки.

В результате выполнения команды была создана зеркальная копия домашней директории пользователя в /tmp/backup без скрытых каталогов:

### Задание 2
- Написать скрипт и настроить задачу на регулярное резервное копирование домашней директории пользователя с помощью rsync и cron.
- Резервная копия должна быть полностью зеркальной
- Резервная копия должна создаваться раз в день, в системном логе должна появляться запись об успешном или неуспешном выполнении операции
- Резервная копия размещается локально, в директории `/tmp/backup`
- На проверку направить файл crontab и скриншот с результатом работы утилиты.

### Выполнение задания 2:
Для выполнения задания был создан bash-скрипт `backup_home.sh`, который выполняет резервное копирование домашней директории пользователя с помощью rsync.

Скрипт создаёт локальную резервную копию в каталоге `/tmp/backup`. Копия является полностью зеркальной, так как используется параметр `--delete`, который удаляет из резервной директории файлы, отсутствующие в исходной директории.

Также в скрипте реализована запись результата выполнения в системный лог:

при успешном выполнении создаётся сообщение `SUCCESS`;
при ошибке создаётся сообщение `ERROR` с кодом возврата.

Содержимое скрипта:

```
#!/bin/bash

# Исходная директория, которую нужно резервировать.
# В данном случае это домашняя директория текущего пользователя.
SOURCE_DIR="$HOME/"

# Каталог, куда будет создаваться локальная резервная копия.
BACKUP_DIR="/tmp/backup"

# Тег для записей в системном логе.
LOG_TAG="backup_home"

# Создаём каталог резервной копии, если он отсутствует.
mkdir -p "$BACKUP_DIR"

# Выполняем зеркальное резервное копирование:
# -a        архивный режим (рекурсивное копирование с сохранением атрибутов)
# --delete  удаляет из резервной копии файлы, которых уже нет в источнике
if rsync -a --delete "$SOURCE_DIR" "$BACKUP_DIR/"; then
    # При успешном выполнении пишем сообщение в системный лог
    logger -t "$LOG_TAG" "SUCCESS: backup of $SOURCE_DIR to $BACKUP_DIR completed successfully"
else
    # Сохраняем код ошибки
    RC=$?

    # При ошибке пишем сообщение в системный лог
    logger -t "$LOG_TAG" "ERROR: backup of $SOURCE_DIR to $BACKUP_DIR failed with exit code $RC"

    # Завершаем скрипт с кодом ошибки
    exit $RC
fi

```
Для автоматического запуска резервного копирования был настроен cron. Задача запускается ежедневно в 02:00.

Содержимое crontab:

```
0 2 * * * /home/USER/backup_home.sh

```

Перед использованием скрипту были выданы права на выполнение:

```
chmod +x /home/USER/backup_home.sh
```

### Вывод:

Таким образом, резервное копирование домашней директории выполняется автоматически один раз в день, создаёт зеркальную копию в `/tmp/backup` и записывает результат выполнения в системный лог.
