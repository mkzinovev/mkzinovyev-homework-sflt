### 5. Настройка Keepalived

На обеих машинах был настроен сервис Keepalived для организации отказоустойчивости с помощью VRRP.

В качестве виртуального IP-адреса был выбран:
```
192.168.56.100/24
```

Этот адрес должен был автоматически переноситься между двумя серверами в зависимости от состояния веб-сервера.

На srv01 Keepalived был настроен как MASTER с приоритетом 150,
на srv02 — как BACKUP с приоритетом 100.

В конфигурации использовалась секция vrrp_script, которая запускала скрипт check_web.sh каждые 3 секунды:

```
vrrp_script chk_web {
    script "/usr/local/bin/check_web.sh"
    interval 3
    timeout 2
    fall 1
    rise 1
    weight -60
}
```

Это означает, что при ошибке выполнения скрипта приоритет узла уменьшается на 60, и виртуальный IP-адрес переходит на второй сервер.

Также были указаны:

1. интерфейс enp0s8
2. одинаковый virtual_router_id
3. одинаковый пароль VRRP
4. unicast-настройки для связи между двумя узлами

Конфигурационные файлы keepalived.conf для серверов srv01 и srv02 с настройкой ролей MASTER и BACKUP, виртуального IP-адреса и секции vrrp_script:
